AI 开发工具对比与选择

技术与工具

深入讨论和比较了多种 AI 编程助手，特别是 Cursor 和 Bolt。Cursor 因其强大的功能（如访问 Claude 3.7/Opus、无限使用、实时预览）而受到推荐，但被批评环境配置复杂、有时安装困难。Bolt 则因其简单易用、UI 友好（尤其是对新手）而受到青睐，但存在 Token 限制和上下文记忆问题。Github Codespaces 被提出作为管理复杂开发环境的解决方案，避免本地配置麻烦。同时，Augment（VSCode 插件，Agent 模式优秀）和 Gemini 2.5 Pro (被认为优于 Claude 3.7) 也被推荐为有力的替代方案。成员根据各自需求和痛点（如 Claude 使用限制）权衡选择。

工具深度对比：Cursor vs Bolt
南阳-成克帅 ~ 四喜-加拿大-_Lv2 ~ 厦门-糖饼 ~ 曲直

**核心观点:**
- **Bolt:** 上手简单，对新手友好，UI 生成效果较好。但 Token 限制严格 (10M 可能半月用完)，上下文记忆弱。
- **Cursor:** 功能强大，可访问 Claude 3.7 (近乎无限)，支持实时预览 (Live Server 插件)。但环境配置是巨大痛点，反复安装令人沮丧，生成的 UI 有时"呆板"，需要调整。
**使用模式:** 有人先用 Bolt 出原型，再用 Cursor 修改；但也有人因环境问题最终放弃 Cursor。
**共识:** Cursor 上限更高，自定义强；Bolt 更易用，但有局限。Cursor 不仅仅是编码工具，也是聊天工具。

用户对 Cursor 和 Bolt 的优缺点进行了详细对比。易用性 (Bolt 胜) 与功能/自由度 (Cursor 胜) 之间的权衡是核心，环境配置是 Cursor 的主要障碍。

杭州-长红 / 厦门-糖饼 / 哈皮 / 曲直 2025-04-15 普遍共识

核心建议是不要过度纠结于初始技术选型：
• **"先完成再完美" (哈皮):** 找到需求、做出 MVP、获得成就感比一开始就追求完美代码更重要。
• **推荐从模板/参考开始 (杭州-长红):** 这样可以专注于业务逻辑，AI 可以辅助参考模板生成。
• **善用 AI 工具 (厦门-糖饼 & 曲直):** 不要低估 Bolt 或 Cursor 的能力，它们可以做出高级网站。推荐新手尝试 Bolt 快速启动。
• **寻找灵感:** Mobbin 和 Github 是寻找模板和开源项目的好地方。
• **SEO 顾虑:** 虽然 AI 代码可能不完美，但可以通过后续优化解决。早期更应关注产品本身。
