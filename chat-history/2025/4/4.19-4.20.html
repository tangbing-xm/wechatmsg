国内域名 (阿里云) 配置 Cloudflare Pages

分享者：灰兔 2025-04-20

介绍了将国内购买的域名（如阿里云）指向部署在 Cloudflare Pages 上的网站的通用方法：核心步骤是在域名注册商（阿里云）的管理后台，找到 DNS 修改设置，将域名的 DNS 服务器 (Nameservers) 地址修改为 Cloudflare 提供的地址。修改完成后，即可在 Cloudflare Pages 项目中添加该自定义域名。此方法适用于任何域名注册商。

核心步骤：

  1. 登录域名注册商后台 (如阿里云万网)。
  2. 找到对应域名的 DNS 修改/管理入口。
  3. 将 DNS 服务器地址 (Nameservers) 修改为 Cloudflare 提供的地址 (通常有2个)。
  4. 等待 DNS 生效 (可能需要几分钟到几小时)。
  5. 在 Cloudflare Pages 项目设置中添加自定义域名。

注意：

  • 国内购买的域名用于国内访问的网站通常需要完成 ICP 备案。
  • 在阿里云后台寻找 DNS 修改入口可能不直观，需要仔细查找或咨询客服。
分类：域名 / DNS / Cloudflare / 部署 / 网站配置 / 阿里云

5. 权限/配置问题： 如果是首次推送或更换了机器/密钥，可能需要检查 SSH Key 是否正确配置并添加到 GitHub 账户。

在阿里云买的域名，如何关联到部署在 Cloudflare Pages 上的网站 (代码在 GitHub)？

域名解析 阿里云 Cloudflare Pages DNS配置 网站部署 备案
哈皮 / 灰兔 2025-04-20 核心步骤

1. 国内域名备案： 首先，确认该阿里云域名是否已完成 ICP 备案（国内服务器访问必须）。
2. 修改 DNS 服务器： 登录【阿里云】的域名管理控制台，找到该域名的 DNS 修改设置，将其 DNS 服务器 (Nameservers) 地址更改为【Cloudflare 提供】的 Nameserver 地址。
3. 等待 DNS 生效： DNS 修改需要一定时间在全球范围内生效，通常几分钟到几小时不等。
4. Cloudflare 添加域名： DNS 生效后，在你的 Cloudflare 账户中添加该域名，并根据指引完成验证。
5. CF Pages 配置： 在 Cloudflare Pages 项目的设置中，找到“自定义域 (Custom domains)”选项，添加你已在 Cloudflare 验证过的域名。
6. 注意： 核心操作与在任何其他注册商（如 Spaceship）购买的域名一致，关键在于修改 DNS 服务器指向 Cloudflare。在阿里云后台找到 DNS 修改入口是关键步骤。
